Windows下简单搭建Hexo博客并部署到Github

发表于 2016-06-08 | 分类于 Hexo

在Windows下简单几步就能拥有属于自己的Hexo博客

环境准备

Git
程序员都应该知道的Git，个人比较喜欢使用GitBash。
Git for Windows

Node.js
在Node.js官网下载Windows版的Node.js，安装时以默认配置一路安装即可

安装Hexo

在任意路径下打开GitBash，输入以下命令：

npm install -g hexo # -g是指全局安装Hexo

初始化Hexo目录

安装Hexo完成后，选择在某文件夹下（例如：F:\Hexo），打开GitBash，输入以下命令：

hexo init # Hexo初始化

安装npm依赖

完成初始化后，接着安装npm依赖

npm install

生成静态页面

hexo generate

本地启动

启动本地服务，对搭建好的博客进行预览调试

hexo server

至此，一个最基本的本地Hexo博客已搭建完成，可以在http://localhost:4000 查看刚刚搭建起来的Hexo博客

部署到Github

首先注册一个Github账号

创建一个与你用户名对于的仓库，仓库名必须形如your_username.github.io

在刚刚创建的仓库下，找到https链接，复制，然后编辑Hexo根目录下的_config.yml，找到文件内deploy，将其中的repository修改成刚刚复制的https链接，修改后保存文件

添加SSH Key（部署Hexo博客到Github时无需再次输入账号密码）

在任意路径下使用GitBash，检查是否已经存在SSH key。如果存在的话，可以直接删除.ssh文件夹下的所有文件

ls -al ~/.ssh

接下来先设置好你注册Github时所填写的邮箱和用户名：

生成密钥（可一直回车）：

ssh-keygen -t rsa -C “your-email”

然后输入以下命令：

ssh-agent -s

接着输入以下命令：

ssh-add ~/.ssh/id_rsa

若提示出错，输入以下命令：

拷贝SSH Key

clip < ~/.ssh/id_rsa.pub

登陆你的Github账号，找到Settings->SSH and GPG keys->New SSH key，将你刚才拷贝的SSH Key粘贴到Key框内，上面的Title可作为这个key的描述

最后测试一下

ssh -T git@github.com

看到Warning的话，输入yes即可

通过以下命令，完成部署

至此Hexo博客已经部署至Github上了，可以在http://your_username.github.io上看到效果

新建文章

输入以下命令，新建一篇名为“postName”的文章：

hexo new “postName”

执行上面的命令后，会生成生成指定名称的markdown文件至 hexo\source\ _posts目录下，使用文本编辑器打开该markdown文件

其中title、date、tags会自动生成，categories等属性需要手动添加。多个tags时要将tag填入[]中，如[tag1,tag2,tag3,…]

在第二个’—‘下开始编写正文

Tips

Hexo命令

ERROR

如果出现以下报错：

ERROR Deployer not found: git

可输入以下命令：

npm install hexo-deployer-git –save
