图解HTTP精华解读

从输入URL到加载页面的过程发生了什么？

  • 读取缓存，如果缓存没过期直接显示
  • 解析DNS
  • 建立TCP连接
  • 发送HTTP请求
  • WEB服务器处理请求并返回（后端程序执行相应代码）
  • 浏览器加载页面
  • 关闭TCP连接

HTTP 协议基础（无状态，持久连接，报文，状态码，Cookie等）

HTTP是一个不保存状态，即无状态的协议。不会对请求和效应的通信状态做持久化处理，如用户登录状态，HTTP不会保存。每次HTTP请求都是独立的。跟TCP协议不同，TCP是保存状态的，发送了多少帧都有序列号和确认号等保存传输状态。

使用cookie解决HTTP无状态的问题

cookie和session区别和联系

  • cookie保存在客户端，session保存在服务器端
  • session安全性更高
  • 保存的数据类型，cookie只能保存字符串
  • cookie保存数据的大小有限制，session无限制
  • sessio会对服务器性能压力更大
  • session依赖于cookie（sessionId），如果禁用cookie，也可以通过url传递

HTTP报文中keep-alive长连接

  • 保存TCP连接一段时间再关闭，复用连接进行多次请求
  • 不改变HTTP请求-响应的过程

确保WEB安全的HTTPS

HTTP的缺点：

  • 通信使用明文（不加密），内容可能被窃听
  • 不验证通信方的身份，因此可能遭遇伪装
  • 无法保证报文的完整性，所以可能被篡改

而HTTPS能够克服上面HTTP的缺点。

在HTTP(应用层) 和TCP(传输层)之间插入一个SSL协议, 就是HTTPS

HTTPS = HTTP+通信加密+证书+完整性保护

websocket技术

  • 全双工通信
  • 连接发起方仍是客户端
  • 减少通信量
  • 使用HTTP建立连接，基于TCP协议

ajax轮询：客户端每隔一段时间发送一次请求，询问服务器是否有新信息

长连接：客户端发起连接后，如果没消息，就一直不返回响应给客户端。直到有消息才返回，返回完之后，客户端再次建立连接，周而复始。

SQL注入和CSRF攻击

SQL注入：通过在表单或URL输入恶意SQL语句，修改数据库内容

防范：不要相信用户任何的输入，对用户的输入进行非法字符的过滤，Mysqli中的预处理是防范SQL注入的一种重要手段

XSS攻击：通过在表单输入恶意JS脚本，当用户浏览该页面时，就会执行恶意JS代码，从而达到攻击用户的目的

防范：对用户输入进行过滤和转义

CSRF攻击：攻击者盗用了你的身份，以你的名义发送恶意请求，对服务器来说这个请求是完全合法的

防范：在请求中加入token进行验证
